# PROJECT CONTEXT: Alexander the Great's Voyage — Interactive Map

---

## 1. PROJECT OVERVIEW

This is a single-file interactive HTML web application that visualizes Alexander the Great's military campaign from 334–323 BC on an animated, interactive Mercator projection map. The project was built entirely in a single file: `alexander_the_great_voyage.html`.

The app has two core functions:
1. **An animated map** that progressively draws Alexander's route segment by segment, using actual historically-sourced GPS waypoints extracted from a Google My Maps KML file.
2. **A narrative sidebar** that displays detailed historical descriptions for each stop on the journey, written in a consistent journalistic style.

The intended audience is general public / non-specialists. The tone is engaging but accurate. The user may want to share this as a GitHub Pages site.

---

## 2. DATA SOURCES

### Primary map data
The route waypoints were extracted from a publicly available Google My Maps titled **"Alexander the Great's Voyage"** (map ID: `1ngCGNBRd3lMT8UylJni6EwHiDbQ`, published September 1, 2020, 81,976 views). The KML data was accessed via:
```
https://www.google.com/maps/d/viewer?mid=1ngCGNBRd3lMT8UylJni6EwHiDbQ
```
The KML contains not just the 19 point markers but the full polyline waypoints for all 18 route segments between them — these are used for smooth, curved animation rather than straight lines.

### Narrative text
The historical descriptions were drafted by the user in a Word document titled **"Alexander the Great draft.docx"**. The draft covered Landing Points 1–12. Landing Points 13–17 and the Final Landing Point were written by Claude in the same style and appended to match. All descriptions are based on historically accurate information cross-referenced with Encyclopædia Britannica.

## 3. FILE STRUCTURE

There is currently **one file** in the project:

```
alexander_the_great_voyage.html   ← entire app, self-contained
```

All CSS, JavaScript, and HTML are in this single file. The only external dependencies are loaded via CDN:
- **Leaflet.js 1.9.4** — map rendering library (CSS + JS)
- **Esri NatGeo World Map tiles** — the basemap (loaded at runtime by the browser)
- Fallback: **OpenStreetMap tiles** (if Esri tiles fail)

No build system, no npm, no frameworks. Pure HTML/CSS/JS.

## 4. MAP SETUP

### Library
**Leaflet.js 1.9.4**, loaded from cdnjs:
```html
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/leaflet/1.9.4/leaflet.min.css"/>
<script src="https://cdnjs.cloudflare.com/ajax/libs/leaflet/1.9.4/leaflet.min.js"></script>
```

### Projection
Leaflet uses the **Web Mercator projection (EPSG:3857)** by default. This is intentional — it's standard for web maps and what was requested.

### Initial view
```javascript
const map = L.map('map', { center: [36, 48], zoom: 4 });
```
This centers the map roughly over the Middle East/Persia, which is the geographic center of Alexander's campaign.

### Tile layer
**Primary:** Esri National Geographic World Map — a vintage atlas-style basemap with terrain, country borders, rivers, and labels. Chosen for its aesthetic match with an ancient history project.
```javascript
L.tileLayer('https://server.arcgisonline.com/ArcGIS/rest/services/NatGeo_World_Map/MapServer/tile/{z}/{y}/{x}', {
  attribution: 'Tiles © Esri — National Geographic, Esri, DeLorme, NAVTEQ',
  maxZoom: 16
})
```
**Fallback:** OpenStreetMap standard tiles (added if NatGeo tiles fire a tileerror event).

## 8. MARKER SYSTEM

All 19 markers are placed on the map immediately at page load, but start in a **dimmed/gray state** to indicate "not yet visited":
```javascript
{ fillColor: '#555', color: '#333', weight: 2, opacity: 0.5, fillOpacity: 0.4 }
```

When a point is reached (via animation), it **lights up** to its active color:
```javascript
// departure: green, landing: gold, final: red
{ fillColor: p.color, color: '#fff', weight: 2.5, opacity: 1, fillOpacity: 0.95 }
```

Marker sizes:
- Departure and Final: radius 10–11px
- All landing points: radius 8–9px

Each marker has a **Leaflet popup** (click to open) showing the point name, date/location, a truncated description (140 chars + ellipsis), and coordinates.

## 9. ANIMATION SYSTEM

### How it works
The `animateSegment(segIdx, onComplete)` function:
1. Takes the raw waypoints for a segment (e.g. 5–20 coordinate pairs)
2. **Interpolates** them into 60 evenly-spaced points using cumulative distance math (so the animation speed is consistent regardless of waypoint spacing)
3. Creates a new Leaflet polyline starting at the first point
4. Uses `setInterval` at 18ms per frame to add one interpolated point at a time via `animLine.addLatLng()`
5. When the last frame is reached, clears the interval, pushes the completed line to `drawnLines[]` array, calls `onComplete()`

```javascript
function animateSegment(segIdx, onComplete) {
  const waypoints = segmentWaypoints[segIdx];
  const STEPS_PER_SEGMENT = 60;
  const interval = 18; // ms
  // ... interpolation + setInterval loop
}
```

### Route line style
```javascript
{ color: '#8B0000', weight: 3, opacity: 0.9, lineJoin: 'round' }
```
Dark red (`#8B0000`), solid (no dash), 3px weight. Chosen over gold/dashed because it shows up clearly against the NatGeo basemap.

### Map fly behavior
After each segment completes, the map flies to the newly revealed destination:
```javascript
map.flyTo([points[currentStep].lat, points[currentStep].lon], 5, { duration: 1.2 });
```
Zoom level 5, 1.2 second flight duration.

### State machine
```javascript
let currentStep = -1;  // -1 = not started
let animating = false; // prevents double-clicks
let drawnLines = [];   // all completed polylines (kept on map)
let animLine = null;   // the currently-animating polyline
```

The `advance()` function handles both buttons:
- If `currentStep === -1`: reveal departure, update sidebar, enable Next, disable Start
- If `currentStep >= 0` and `segIdx < 18`: animate segment → light up marker → show info → update progress → fly map
- If `currentStep === 18` (final): disable Next, change text to "— Complete —"

## 10. PROGRESS BAR

Width is computed as:
```javascript
const pct = currentStep < 0 ? 0 : Math.round((currentStep / (points.length - 1)) * 100);
document.getElementById('progress-fill').style.width = pct + '%';
```
`points.length - 1 = 18`, so each step advances by ~5.6%.

CSS transition: `width 0.5s ease` for smooth fill animation.

## 13. KNOWN ISSUES / THINGS TO WATCH OUT FOR

1. **Esri NatGeo tiles cap at zoom 16** — if users zoom in very far, tiles stop loading. This is a tile provider limitation.
2. **The tileerror fallback** (OSM) is registered but OSM tiles look quite different from NatGeo. If NatGeo goes down, the visual style changes noticeably.
3. **No mobile responsiveness** — the sidebar is fixed at 320px. On small screens it may crowd the map. The layout hasn't been optimized for mobile.
4. **No "Back" / "Restart" button** — once you advance past a point, there's no way to go back without refreshing the page.
5. **Animation can't be interrupted** — during the `setInterval` animation, the Next button is disabled. If the user clicks very rapidly before the disable takes effect, `advance()` has an `if (animating) return` guard.
6. **Segment 11 (LP11 → LP12)** has only 2 waypoints (it's a very short segment). The interpolation still works but the line is nearly straight.
